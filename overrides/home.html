Contribute a part

Add a part in one command.

The catalog is AI-authored end‑to‑end — so contributing is too. Point an agent at the repository and run a single command with your part. It sources the sequence from a cited deposit (independent re-fetch + compare, never from memory), researches the literature, annotates sub-features with Sequence Ontology, and drafts functional claims — each carrying its source, confidence, and review status — then runs every validation gate and opens a pull request from your fork. It is proposal-only: nothing lands until you review it.

1

Fork the catalog & open it

You don't need write access — you work from your own fork. The GitHub CLI forks, clones, and wires up upstream in one line; then open Claude Code. (Repo developers can git clone directly.)

$gh repo fork dbikard/seqmake-parts --clone && cd seqmake-parts && claude

One-time setup: a GitHub account, gh auth login, and Claude Code.

2

Run one command — it does the rest

Describe your part in plain words. The engine sources, researches, annotates and verifies, runs every gate, then pushes a branch to your fork and opens the pull request to the catalog. It's proposal-only — you review before it lands.

/add-part GFPmut3
→sourced · verified · gates green · PR → dbikard/seqmake-parts

What counts as proper input

Just describe it in plain words — the agent figures out the rest. No special format.

Just the part name — the engine finds the canonical source:
/add-part T7lac
Name + its source or paper — an accession, PMID or DOI, however you'd say it (most robust):
/add-part aadA, from UniProt P0AG05 (see PMID 26527143)
A whole related family at once — researched once, grouped together:
/add-part the Anderson promoters J23100, J23101, J23102

Helpful, never required — more provenance means a better, less-provisional result. If the defining paper is paywalled, drop the PDF in sourcing/incoming/ and re-run: the agent verifies and cites boundaries from it, and the PDF stays local (it's gitignored — never committed; only the citation lands).

What the agent does for you

  • Sources the sequence by independent re-fetch + byte/alignment compare against the deposited record — flags any divergence, never trusts memory.
  • Types every feature with Sequence Ontology and locates sub-features (−35/−10, operator, RBS, domains) on the real sequence, then adversarially verifies the coordinates.
  • Drafts literature-cited functional claims that land as ai-generated, ready to move along the pipeline.
  • Runs every gate — schema, GenBank/catalog/RDF rebuild, SHACL, content guard, tests — then pushes to your fork and opens the PR to the catalog only when they are green.
